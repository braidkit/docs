# Event Stories

Event stories explain the protocol by walking through concrete work.

Start with these:

- [Human thread](../stories/human-thread.md): a human edits code and records a
  decision.
- [Agent thread](../stories/agent-thread.md): an agent receives a prompt, uses
  tools, commits, and decides.
- [Sub-agent delegation](../stories/sub-agent-delegation.md): a parent agent
  delegates part of an attempt and rolls up the result.
- [Review and promote](../stories/review-and-promote.md): a reviewer approves
  work and the orchestrator promotes it.

The point of each story is not just event order. It is to show what the user is
doing and which event helps future reviewers understand that moment.
